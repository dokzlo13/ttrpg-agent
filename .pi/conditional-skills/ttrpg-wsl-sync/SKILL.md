# ttrpg-wsl-sync

Local WSL/Windows mirror workflow for this project. This skill is exposed only
when `TTRPG_WINDOWS_AGENT_DIR` is configured. It never contains the user's
private mirror path; the path comes from ignored `.env` or the process
environment.

## Source boundaries

- WSL project root (`$TTRPG_ROOT`, usually current git checkout) is the primary
  source of truth for generated/ingested library content.
- Windows mirror (`$TTRPG_WINDOWS_AGENT_DIR`) is for Obsidian editing and raw
  import drop-off.
- Pulling from Windows defaults to **active notes only**.
- Pulling ingested books from Windows is available, but must be explicit because
  `vault/library/books/` is normally generated by `book-ingest`.
- Imports sync is one-way, new-files-only, and only copies the raw books inbox
  (`imports/books/`). It does not touch local `imports/5etools/`,
  `imports/source-vault/`, qmd state, or generated vault library output.

## Commands

Run from the project root unless noted otherwise.

```bash
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh status
```

### Push WSL vault to Windows mirror

Use after local/agent changes when Windows Obsidian should see the latest vault.
This is the only mode that uses `--delete`, and only against the Windows mirror.
Always dry-run first unless the user has explicitly asked to apply.

```bash
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh push-vault --dry-run
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh push-vault --apply
```

### Pull Windows note edits back into WSL

Default pull strategy. This copies `Windows/vault/notes/` into `WSL/vault/notes/`,
backs up overwritten WSL files into `.trash/`, then runs `qmd update && qmd embed`
and verifies with a follow-up rsync dry-run.

```bash
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh pull-notes --dry-run
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh pull-notes --apply
```

### Pull edited ingested books back into WSL — explicit only

Use only when the user says they edited generated/ingested book Markdown on the
Windows side and wants those edits preserved in WSL. This copies
`Windows/vault/library/books/` into `WSL/vault/library/books/`, backs up
overwritten WSL files into `.trash/`, then reindexes and verifies. It does not
use `--delete`.

```bash
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh pull-books --dry-run
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh pull-books --apply
```

### Pull new raw books from Windows imports inbox

New-files-only. No overwrite, no delete.

```bash
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh pull-imports-inbox --dry-run
.pi/conditional-skills/ttrpg-wsl-sync/scripts/sync.sh pull-imports-inbox --apply
```

After applying imports, list copied files and ask whether the user wants to
run the normal `ttrpg-import-book-pdf` ingest workflow.

## Safety rules for agents

1. Use `status` first if path/config state is unclear.
2. Use dry-run before `--apply` unless the user explicitly asked to apply.
3. Do not manually edit `vault/library/books/`; `pull-books` is only for
   preserving explicit Windows-side user edits.
4. Do not add ad-hoc rsyncs for `imports/5etools/`, `imports/source-vault/`,
   `.qmd/`, `.pi/`, or repo files.
5. After `pull-notes --apply` or `pull-books --apply`, check the script's qmd and
   verification output before reporting success.
