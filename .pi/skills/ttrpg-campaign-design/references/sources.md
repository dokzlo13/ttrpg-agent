# Method sources

This skill uses independently written, project-specific guidance informed by the
following methods. No upstream code, templates, CLI, or directory schema is
installed with it.

## Tabletop methods

- Justin Alexander, “Don't Prep Plots”
  https://thealexandrian.net/wordpress/4147/roleplaying-games/dont-prep-plots
- Justin Alexander, “Tools, Not Contingencies”
  https://thealexandrian.net/wordpress/37422/roleplaying-games/dont-prep-plots-tools-not-contingencies
- Justin Alexander, “Node-Based Scenario Design”
  https://thealexandrian.net/wordpress/8122/roleplaying-games/node-based-scenario-design-collectors-edition
- Justin Alexander, “The Three Clue Rule”
  https://thealexandrian.net/wordpress/1118/roleplaying-games/three-clue-rule
- Justin Alexander, “Using Revelation Lists”
  https://thealexandrian.net/wordpress/40978/roleplaying-games/random-gm-tip-using-revelation-lists
- Blades in the Dark, “Progress Clocks”
  https://bladesinthedark.com/progress-clocks

Adapted ideas: present-tense situations, actor tools instead of contingencies,
node reachability, resilient revelations, proactive information, and clocks as
fictional-state records rather than automatic schedules.

## Agent-skill donors reviewed but not installed

- `danjdewhurst/story-skills`, revision `c482d48f4eb9`, MIT
- `adaumann/speckit-preset-game-narrative-writing`, revision `0285ed9d36e7`, MIT
- `MRCalderon3D/everything-game-dev-code`, revision `9585a1c63973`, MIT

Retained only as general ideas: explicit state distinctions, knowledge
asymmetry, conditional information transmission through established
communication, promise lifecycle, discoverable traces for delayed consequences,
affected-stakeholder legibility without forced closure, convergence auditing for
erased consequential differences, and semantic review before state promotion.

Rejected: novel/chapter schemas, act and climax structures, fixed endings,
branch/variable simulation, trust scores, software-game UI/telemetry/QA concerns,
and deterministic story CLIs.
